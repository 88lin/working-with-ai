When are Skills useful?
Help AI quickly learn specialized tools · domains you're unfamiliar with
Even without skills, agents can learn from docs — skills are just distilled indexes
When are Skills less useful?
Domains the model already knows well (mainstream languages/frameworks) · may limit AI flexibility
After model updates, skills can become bottlenecks — following old patterns when the model already "knows better"
Worth exploring: skills npm package ↗ · skills.sh ↗
CLI vs. MCP
CLI
👤 Human / Agent → 💻 Shell → 🔧 Tool
Fast Composable Transparent
Inner dev loop · Agent calls directly via shell
+
MCP
🤖 Model → 📡 Protocol → 🔧 Tool
Discoverable Cross-client Structured I/O
Outer integration loop · Standardized tool discovery & permissions
For agents, both are valuable — most teams end up using both
CLI for low-level implementation + expert access, MCP for standardized interface layer
Recommended: gh CLI

Personal Prompt Tips
01
Think from first principles
Don't apply patterns blindly, derive from fundamentals
02
Fix at the root, avoid ad-hoc patches
Find the essence of the problem, don't just band-aid it
03
Stay neutral about my approach
Don't assume my previous approach was correct — challenge when needed
04
Understand existing code before changing it
Don't start editing before you've finished reading — understand why it's written this way
05
Trust AI's foundational capabilities
Don't specify file paths or fixed context — let AI find and judge on its own
Use it a lot, build a complete sense of what AI can and can't do
For what AI excels at → trust AI

A single search might be faster, but overall time to answer is faster with LLM
My Google usage dropped 95%

Default Distrust of "Fast"
After using AI, I actually started trusting "slow"
"Fast" gives me an uneasy feeling — did AI skip something?
"Slow" means it's thinking carefully, researching, planning
It's the same intuition we have about people:
Would you trust someone who answers in 3 seconds?
